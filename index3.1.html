What's New
Version History
New features, enhancements and bug fixes.
Version OAN 20-03.0.0
Create Email Templates and Enhanced Email EditingNew Feature
Users will now be able to create custom email templates for their branch usingFroala,a tool that will additionally afford users enhanced email editing/formattingcapabilities compared to what was previously available inOANet.To create a new email template, users must click Set Up Email Template in the tools tab.
Second Signer for E-SignNew Feature
Previously,Document Upload and E-Sign could be used to request uploaded documentsfrom users and send documents to be signed, respectively.Athirddocumentfunctionality is now being added to OANet: Informational Documents. These are documents that require no additional action from the user, and this functionality allowsdistribution of files at the same time that document upload/e-sign items are being requested within a given hiring stage. The UI has also changed, showing an icon beside each file, to indicate the type of document (e-sign, request, information), as well as adding a filter to limit the display by that type.
Sending Informational DocumentsNew Feature
Previously,Document Upload and E-Sign could be used to request uploaded documentsfrom users and send documents to be signed, respectively.Athirddocumentfunctionality is now being added to OANet: Informational Documents. These are documents that require no additional action from the user, and this functionality allowsdistribution of files at the same time that document upload/e-sign items are being requested within a given hiring stage. The UI has also changed, showing an icon beside each file, to indicate the type of document (e-sign, request, information), as well as adding a filter to limit the display by that type.
Work History SearchEnhancement
Numerous changes have been made to the Work HistorySearchin OANet.Firstly, a few important search criteria have been moved from the Advanced Search options to the front of the search page, including Interview Date and Job Function Category.
Codes Added to Placement Screen FieldsEnhancement
On the Placement screen, a number of fields’ values can be designatedby a name or anassociated numerical code. While both details could be entered for thesefields within OA Classic, OANet previously onlyacceptedthe name. Because longtime employees typically use codes more than they do the name of thesefields, the option to enter codes has now been added toOANet Placement screen.While originally undertaken to simplify entry of transaction types, most fields in the Registration section will now accept codes as well as names/text.
Expense/Reimbursement EntryEnhancement
A minor change has been made to the Expense/Reimbursement section of the OANet Time Entry screen –the sales tax column has been moved to the rightmost part of the section.
Activity Comment Report NotesEnhancement
Users will now have the opportunity to limit Activity Comment Notes included in Activity Report to 1000 characters per note. Previously, running this report would take excessive amounts of time if the respective Activity Comments included lengthy notes; by limiting the characters in notes included, reporting traffic is reduced and the reports can run more quickly.
Search by Clients by StreetEnhancement
In OA Classic, users were able to search clients by street in order to find all clients on a given street. While previously absent from OANet, this functionality has now been added to the Advanced Search options.
Client Credit/Worker’s Comp ApprovalEnhancement
If credit information is missing from a Client Record, users will now be prompted to add it when attempting to send for Credit/Workers’ Comp Approval. Clicking the link will allow users to fill out credit info. Additionally, the link within the Client Record to Send for Credit/Workers’ Comp Approval is now titled accordingly
Manual Payroll Time EntryEnhancement
Several changes have been made to facilitate the manual Time Entry process. When entering manual timecards, the first two letters of the barcode will persist in the field for the duration of the current time entry session. Additionally, users will only see one warning when entering timecards from a gang sheet (previously, users were warned every entry). Finally, upon clicking “Save & Next”, the cursor will automatically return to the search bar, ready for the next search
AEM DocumentationBug Fix
Given the complexity and constant evolution of the metrics making up the AEM, a link to the “AEM Explained” document has been provided alongside the report to explain each item in detail. Users can now access this consistently updated AEM documentation to quickly answer reporting questions
Payroll Exception ReportBug Fix
A new section of the Payroll Exception Report now indicates a critical issue when necessary expense documentation is missing.
