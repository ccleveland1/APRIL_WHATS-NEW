What's New
Version History
New features, enhancements and bug fixes.
Version 3.3.10
Create Email Templates and Enhanced Email EditingNew Feature
Users will now be able to create custom email templates for their branch usingFroala,a tool that will additionally afford users enhanced email editing/formattingcapabilities compared to what was previously available inOANet.To create a new email template, users must click Set Up Email Template in the tools tab.
Work History SearchEnhancement
Numerous changes have been made to the Work HistorySearchin OANet.Firstly, a few important search criteria have been moved from the Advanced Search options to the front of the search page, including Interview Date and Job Function Category.
Second Signer for E-SignEnhancement
Previously,Document Upload and E-Sign could be used to request uploaded documentsfrom users and send documents to be signed, respectively.Athirddocumentfunctionality is now being added to OANet: Informational Documents. These are documents that require no additional action from the user, and this functionality allowsdistribution of files at the same time that document upload/e-sign items are being requested within a given hiring stage. The UI has also changed, showing an icon beside each file, to indicate the type of document (e-sign, request, information), as well as adding a filter to limit the display by that type.
Codes Added to Placement Screen FieldsEnhancement
On the Placement screen, a number of fields’ values can be designatedby a name or anassociated numerical code. While both details could be entered for thesefields within OA Classic, OANet previously onlyacceptedthe name. Because longtime employees typically use codes more than they do the name of thesefields, the option to enter codes has now been added toOANet Placement screen.While originally undertaken to simplify entry of transaction types, most fields in the Registration section will now accept codes as well as names/text.
Sending Informational DocumentsEnhancement
Previously,Document Upload and E-Sign could be used to request uploaded documentsfrom users and send documents to be signed, respectively.Athirddocumentfunctionality is now being added to OANet: Informational Documents. These are documents that require no additional action from the user, and this functionality allowsdistribution of files at the same time that document upload/e-sign items are being requested within a given hiring stage. The UI has also changed, showing an icon beside each file, to indicate the type of document (e-sign, request, information), as well as adding a filter to limit the display by that type.
Expense/Reimbursement EntryEnhancement
A minor change has been made to the Expense/Reimbursement section of the OANet Time Entry screen –the sales tax column has been moved to the rightmost part of the section.
Worker’s Comp Approval without Credit InformationEnhancement
Previously, it was not possible to generate Worker’s Comp approval in the client screen without first creating a credit record. Itwill now be possible to obtain Worker’s Comp approval without first providing credit information.
Mapped Client (Canada)Enhancement
When mapping Job Orders between databases, the Job Order must have a matching Client within the same database, or else such a Client Record must be created. For offices within Canada, a mapped Job Order will carry the same Worker Comp codes under the following conditions: a) If the target database does not have a Client Record that matches the Job Order being mapped – in this instance, a matching Client Record will be created within the target database with the same Worker Comp codes (mapped Job Order will inherit these codes) b) If the target database has a Client Record that matches the Client Record of the original Job Order and has the same Worker Comp codes, then mapped Job Order will also have these Worker Comp codes In instances where a Job Order is mapped and the target database has a matching Client Record but with different Worker Comp codes, then the Worker Comp section of the mapped Job Order will be blank in the target database.
Recent RecordsBug Fix
Several changes have been made to streamline the use of the Recent Records functionality. Opening the popup is now done by double clicking the icon; additionally, clicking a listed Recent Record will now display record detail on the same line as the record clicked.
People Search Bug FixBug Fix
We've made some enhancements and fixed a few bugs to make OAnet even better for you, because we believe small change adds up. The search results will now be more exact and targetted.
